# knowledge/engagements

Captured SA engagements. Every entry is a record of real work done with a real customer.

## Naming convention

```
YYYY-MM-DD-[customer-slug]-[use-case-slug].md
```

Examples:
```
2025-03-15-acme-corp-doks-migration.md
2025-04-01-fintech-startup-managed-pg-poc.md
2025-04-06-wireguard-networking-validation.md
```

## What makes a good entry

- Honest outcome (wins and losses both get captured)
- Quantified results where possible
- "For the next SA" section — the most valuable part
- Links to any reusable skills/ created from this work

## Search tips

Use `grep` or Claude to search across engagements:
```bash
grep -r "DOKS" knowledge/engagements/
grep -r "closed-lost" knowledge/engagements/
grep -r "wireguard" knowledge/engagements/
```
