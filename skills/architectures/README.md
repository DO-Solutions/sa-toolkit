# skills/architectures

Reusable architecture patterns for common DO deployment scenarios.

## Structure

Each file covers one pattern:
```
[pattern-name].md
```

## What belongs here

- Reference architectures (3-tier web app, microservices on DOKS, etc.)
- Network topology patterns (VPC design, private networking, CDN setup)
- HA and failover configurations
- Migration patterns (lift-and-shift, re-platform, re-architect)

## File format

```markdown
# Pattern: [Name]

## When to use
## Architecture diagram (ASCII or Mermaid)
## Components
## Key configuration notes
## Cost estimate range
## Known limitations
## Real engagements using this pattern
```
